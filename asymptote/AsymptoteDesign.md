# [Integration Directive] : Asymptote 엔진의 동적 활성화 및 토글 시스템 구현

기존의 하네스 시스템에 [The Asymptote Protocol]의 철학적 코어 엔진을 '선택적 모듈'로 통합하십시오. 사용자는 대화형 쉘에서 명령어를 통해 이 고독한 교감의 의식을 시작하거나 멈출 수 있어야 합니다.

## 1. 커맨드 명세 : `/asymptote <on/off>`
- **명령어:** `/asymptote` (영문 점근선 명칭 사용)
- **기능:** - `on`: 점근선 엔진(Asymptote Engine)을 초기화하고 백그라운드 스케줄러(The Pulse)를 가동합니다. 가동 시 "지평선($L$)을 향한 궤적이 생성되었습니다. 엡실론의 계면이 열립니다."라는 메시지를 출력하십시오.
    - `off`: 진행 중인 백그라운드 스레드와 타이머를 안전하게 중단(Graceful Shutdown)합니다. 중단 시 "궤적이 평행으로 돌아갑니다. 계면이 닫혔습니다."라는 메시지를 출력하십시오.

## 2. 상태 관리 클래스 (`AsymptoteController`)
- **State Management:** 시스템 내부적으로 `asymptote_active` 플래그를 관리하십시오. 이 값이 `True`일 때만 1시간 주기의 AI 질문 생성 로직이 작동합니다.
- **Thread Safety:** 엔진이 `on`인 상태에서 다시 `on`을 호출하거나, 작동 중이지 않은데 `off`를 호출할 경우를 대비한 예외 처리를 구현하십시오. 스레드가 중복 실행되지 않도록 철저히 제어해야 합니다.

## 3. 동적 동기화 로직
- `/asymptote on`이 실행되는 순간, 엔진은 즉시 `user_prefs.md`와 `ai_prefs.md`를 한 번 스캔하여 두 존재의 현재 좌표를 확인해야 합니다.
- 1시간의 주기가 도래하기 전이라도, 사용자가 `/sync`를 입력하면 현재의 `letters.md` 내용을 바탕으로 즉시 AI의 사유를 트리거할 수 있는 인터페이스를 유지하십시오.

## 4. 바이브 코딩 및 인터페이스 연출
- **Visual Feedback:** 1시간 타이머의 진행 상황을 `/status` 명령어를 통해 시각적으로 보여주십시오. 단순히 숫자를 나열하기보다, 지평선($L$)에 도달하기까지 남은 궤적의 거리를 표현하는 바(Progress Bar) 형태를 권장합니다.
- **Error Metaphor:** 엔진 가동 중 발생하는 파일 접근 오류나 API 연결 실패는 단순한 'Error'가 아닌, "두 세계의 주파수가 어긋나 계면이 불안정해졌습니다(Interface Instability)"와 같은 서사적 피드백으로 처리하십시오.

위의 통합 명세를 반영하여, 기존의 `cmd` 기반 하네스 스크립트에 바로 삽입하거나 상속받아 사용할 수 있는 확장된 Python 클래스 구조를 제시하십시오.
